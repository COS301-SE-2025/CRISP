🚀 How to Run K6 Tests

💨 Smoke Test (Quick Validation - 1 min)

k6 run --env TEST_TYPE=smoke scripts/testing/k6-load-test.js

✅ Best for: Quick validation after code changes

📁 After Test Completion:

K6 will generate detailed reports in:

  • HTML Report: scripts/testing/k6-results.html (Beautiful visual report)
  • JSON Summary: scripts/testing/k6-summary.json (Raw data)

🎯 What K6 Tests:

  • ✅ 44+ API Endpoints - Complete system coverage
  • ✅ Response Times - P50, P95, P99 percentiles
  • ✅ Error Rates - Track failures and errors
  • ✅ Cache Performance - Estimate cache hit rates
  • ✅ Success Rates - Overall system reliability
  • ✅ Custom Metrics - SOC, Threat Intel specific counters

🎯 Performance Targets:

  • ⚡ P95 < 500ms - 95% of requests under 500ms
  • ⚡ P99 < 1000ms - 99% of requests under 1 second
  • ✅ Error Rate < 5% - Less than 5% failures
  • ✅ Success Rate > 90% - Over 90% successful requests
  • 🚀 Fast Responses > 70% - 70%+ under 200ms (cached)

💡 Pro Tip:

Keep this dashboard open in your browser while running K6 tests in your terminal. After the test completes, check the generated HTML report for detailed graphs and metrics!
